# BeeGenius

Сайт про бджільництво "Bee Genius · Валентин" — блог, наукові статті, оголошення, проєкти. Власна легка адмінка (PHP, без CMS/фреймворку), вхід через Google. Детальний опис архітектури й поточного стану — у `beegenius-plan.md`. Посібник для власника сайту (як користуватися адмінкою, без технічних термінів) — у `POSIBNYK.md`.

## Як подивитись локально

Через XAMPP: проєкт лежить у `C:\xampp\htdocs\beegenius`, відкрити `http://localhost/beegenius/public/`.

Або вбудованим сервером PHP:
```
cd public
php -S localhost:8000
```
Потім відкрити http://localhost:8000

Перед першим запуском: скопіювати `config/config.example.php` у `config/config.php`, заповнити дані MySQL і (за потреби) Google OAuth. Застосувати `db/schema.sql` і всі файли з `db/migrations/` по черзі через phpMyAdmin.

## Структура

```
/app/controllers        — маршрути/логіка публічних сторінок
  /admin                 — контролери адмінки
/app/views               — HTML-шаблони (PHP)
/app/lib                 — db.php, helpers.php, auth.php, csrf.php, sanitize.php, tags.php
/config/config.example.php — скопіювати в config.php і заповнити реальними даними (config.php у .gitignore)
/db/schema.sql           — довідковий знімок схеми
/db/migrations/          — послідовні .sql-міграції (застосовувати по черзі, вручну)
/public                  — точка входу (index.php), assets (css/img/fonts)
/public/uploads/articles — фото від адмінки (поза git; попри назву — сюди йдуть усі фото: статті, проєкти, відгуки, хіро-банер, картки "Чим я займаюся")
```

`app/`, `db/`, `config/`, `tmp/` мають власні `.htaccess` (`Require all denied`) — захист про всяк випадок, якщо document root хостингу колись вкаже не на `public/`, а на корінь проєкту.

## Чисте розгортання (новий сайт, без наявних даних)

Якщо ставите сайт на новий сервер з нуля — без перенесення наявної бази.

**1. Файли.** Скопіювати проєкт (`app`, `public`, `config`, `db`) на сервер, наприклад через `git clone`. Тека `public/uploads` і `config/config.php` в git немає — створюються окремо (нижче).

**2. База даних.** Створити нову порожню базу MySQL. Застосувати **лише** `db/schema.sql` — він містить актуальну структуру одразу з усіма таблицями (`articles`, `tags`, `article_tags`, `quotes`, `projects`, `site_settings`, `activities`, `admin_login_log`) і не потребує додатково файлів з `db/migrations/` (ті — тільки для покрокового оновлення вже наявної, раніше засіяної бази, тут не застосовні). За бажання — додатково `db/seed.sql` для прикладового стартового контенту.

**3. Налаштування.** Скопіювати `config/config.example.php` → `config/config.php`, заповнити:
- дані нової MySQL-бази;
- `google_oauth.client_id` / `client_secret` / `redirect_uri` — після реєстрації застосунку в Google Cloud Console (OAuth consent screen + Client ID);
- `admin_whitelist` — email того, хто матиме доступ до адмінки (прибрати `test-admin@local.dev`, він лише для локальної заглушки `/dev-login`, яку на проді використовувати не варто — вона доступна лише коли `IS_LOCAL`, тобто на `localhost`/`127.0.0.1`, тож на реальному домені й так неактивна).

**4. Тека для завантажень.** Створити `public/uploads/articles/` з правом на запис для веб-сервера (туди зберігаються всі фото, завантажені через адмінку).

**5. Розташування.** Document root хостингу має вказувати на теку `public` — тоді сайт відкривається в корені домену, без `/public` у адресі.

**6. Перший вхід.** Зайти на `/login`, увійти Google-акаунтом з `admin_whitelist` → відкриється `/admin`. Заповнити контакти (`/admin/settings`) і головну сторінку — тексти, хіро-фото, картки "Чим я займаюся" (`/admin/home`), і, за потреби, перший проєкт/статтю. Щоб на головній зʼявився лічильник — позначити один проєкт прапорцем "Показувати лічильник на головній" (форма проєкту) і опублікувати його.

## Перенесення сайту на інший хостинг (з наявними даними)

Три частини: файли, база даних, налаштування.

**1. Файли.** Скопіювати весь проєкт (`app`, `public`, `config`, `db`) на новий хостинг. Обов'язково — теку `public/uploads` (фото завантажені через адмінку), вона поза git і не переноситься автоматично разом з кодом.

**2. База даних.** У phpMyAdmin на старому хостингу: "Экспорт" → вибрати всі таблиці → формат SQL → "Структура + дані" → скачати `.sql`. На новому хостингу створити нову базу MySQL і зробити "Импорт" цього файлу — так переноситься все: статті, теги, проєкти, відгуки, контакти, лічильники.

**3. Налаштування (`config/config.php`).** Цей файл у `.gitignore`, тому переноситься тільки вручну — скопіювати окремо і відредагувати під нове середовище:
- `db.host` / `db.name` / `db.user` / `db.pass` — дані нової бази.
- `google_oauth.redirect_uri` — новий домен (`https://новийдомен.com/auth/callback`), і оновити цю ж адресу в Google Cloud Console → OAuth consent screen.
- `admin_whitelist` — прибрати `test-admin@local.dev` (лише для локальної заглушки `/dev-login`), лишити реальні email адміністраторів.

**4. Розташування.** На проді сайт має бути в корені домену (document root → тека `public`) — тоді `BASE_PATH` автоматично стає порожнім і всі адреси йдуть без `/beegenius/public`.

**Після перенесення перевірити:** вхід через Google (новий redirect_uri), завантаження фото в статтях (права на запис у `public/uploads/articles`), `/sitemap.xml` і `/robots.txt` показують правильний домен (вони генеруються динамічно з поточного хоста, налаштовувати вручну не треба).
